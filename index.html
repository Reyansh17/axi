01 · About AXI
The definition

AXI = Advanced eXtensible Interface

A high-performance on-chip bus protocol, part of the AMBA family. Designed from the ground up for one goal: let fast, complex SoCs move data efficiently without the bottlenecks of older buses like AHB or APB.

“
The AXI protocol supports high-performance, high-frequency system designs for communication between Manager and Subordinate components. — AMBA AXI
01 · About AXI
Where AXI fits

The problem: address-phase lock

Old buses — AHB / APB

Every transfer ties up the bus until its data comes back. Address and data are locked together.

02 · Architecture
Notice the asymmetry

Writes use 3 channels, reads use 2

WRITE — three channels

AW W B

Address and data travel independently, so a dedicated B channel signals completion after all write data beats arrive.

READ — two channels

AR R

Data returns on R carrying its own response code inline — no separate response channel needed.

⚠ Critical rule B fires once per transaction, not per beat. A 16-beat write transfers 16 W beats, but B fires exactly once — after the last beat. Fundamentally unlike AHB, where every transfer had its own response phase.

02 · Architecture
Precise terminology — not interchangeable

Transfer vs Transaction

Transfer

communication in one clock cycle on one channel (one VALID+READY handshake)

Transaction

the complete set of transfers for one Manager↔Subordinate operation

A 4-beat burst write =
1 AW+ 4 W+ 1 B= 6 transfers · 1 transaction

A protocol checker fires on transfer-level violations; performance analysis works at transaction granularity.

Key concepts
that make it fast

Three ideas: separate phases, bursts, and outstanding transactions.

03 · Key Concepts
Concept 1

Separate address & data phases

AHB — coupled
addrdata…wait…
addrdata

Bus occupied for the whole transaction. Next one can't start until this finishes.

AXI — decoupled
ARa1a2a3
Rd1d2d3

Issue a3 while d1 is still in flight. Independent flow control per channel.

Why it matters — a DRAM controller may take 80 cycles per request. Without separation, everything upstream stalls for all 80. With AXI, the Manager fires the next address immediately, hiding latency behind useful work.
03 · Key Concepts
Concept 2 · try the sliders

Burst-based transfers

Send one start address; the Subordinate computes the rest from three fields.

AxBURST address mode
8 beats × 8 B = 64 bytesFIXED — every beat hits the same address
Instead of 16 addresses for a 16-beat burst, you send one. Address-channel bandwidth is a tiny fraction of data bandwidth — which is why interconnects optimize for data throughput.
03 · Key Concepts
Concept 3

Multiple outstanding transactions

📦 The postal model

Drop off 5 packages at once. Each gets a tracking number (transaction ID). Deliveries come back in whatever order the routes allow — you match tracking numbers to know which arrived.

ID 1 ↗
ID 2 ↗
ID 3 ↗
↘ ID 2
↘ ID 1

IDs: AWID/ARID on the way out, BID/RID on the way back.

The ordering rule

✓
Different IDs

may complete out of order — hardware exploits parallelism

=
Same ID

must complete in issue order — software enforces ordering
